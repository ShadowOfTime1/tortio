# Changelog

Все значимые изменения по версиям. Формат — обратный хронологический.

## v1.13.0 — 2026-04-24

- **Hero-фото в шапке экрана пересчёта.** Если у рецепта есть фото, оранжевый градиент в верхней части `scaler_screen` заменяется самой картинкой с тёмным overlay (45%), чтобы белые иконки и текст оставались читаемыми.
- **Drag-and-drop секций** в форме рецепта — у каждой секции в заголовке появилась иконка ⋮⋮, long-press которой запускает перетаскивание.
- **Drag-and-drop ингредиентов** внутри секции — та же иконка слева от поля «Ингредиент». Можно переставлять порядок ингредиентов внутри одной секции.
- **`RadioGroup` миграция.** Переписал диалог «Свой тип секции» с устаревшего `RadioListTile.groupValue/onChanged` на современный `RadioGroup` ancestor (Flutter 3.32+ API). Шесть info-предупреждений в analyze исчезли.

## v1.12.2 — 2026-04-23

- **Hotfix:** заменил `file_picker` на `file_selector` (официальный Flutter-плагин), чтобы убрать неразрешимый конфликт `win32` версий между `file_picker`, `share_plus` и `package_info_plus`. Импорт работает идентично — open dialog с фильтром по `.json`.

## v1.12.1 — 2026-04-23 (release не опубликован, см. v1.12.2)

- Hotfix `file_picker` namespace, который сломал win32-зависимости.

## v1.12.0 — 2026-04-23 (release не опубликован, см. v1.12.2)

- **Теги/категории.** В рецепте — поле тегов через запятую (например, «шоколадный, без глютена»). В списке появляется лента-фильтр по тегам, всегда есть кнопка «Все» для сброса.
- **Кастомные типы секций.** В пикере секций — кнопка «+ Свой тип»: вводишь название, эмодзи, выбираешь режим масштаба (объём/площадь/фикс). Сохраняется отдельно от пресетов.
- **Экспорт / импорт коллекции.** В меню «⋮» в шапке: «Экспортировать» собирает JSON всех рецептов и кидает в системный share-диалог; «Импортировать» открывает file-picker, читает JSON, добавляет к существующим. Картинки в JSON не передаются (пути локальные).
- **Drag & drop пересортировки рецептов.** Long-press на карточку → перетаскивание. Работает только когда фильтры (поиск/тег) выключены — иначе сортировка по видимому списку не отражается на хранилище. Drag для секций и ингредиентов внутри рецепта — отложен (нужны nested scrollable аккуратные).
- **Автокомплит ингредиентов.** Поле имени ингредиента превратилось в `Autocomplete` с подсказками из истории — все когда-либо введённые названия отсортированы по частоте.
- **Фото/обложка рецепта.** В форме сверху — круглый аватар, тап открывает выбор «Из галереи / Сделать фото / Убрать». Картинка копируется в `applicationDocuments` (переживает рестарты). В списке миниатюра заменяет дефолтный 🍰. На экране пересчёта пока без hero-фото — будет в следующем релизе.

## v1.11.0 — 2026-04-23

- **Дублирование рецепта.** В меню «⋮» рядом с рецептом появилась команда «Дублировать» — создаёт копию `{title} (копия)` со всем содержимым.
- **Undo при удалении.** Вместо подтверждающего диалога — прямое удаление со SnackBar «Отменить» (4 секунды). Меньше тапов, но безопасно.
- **Меню действий вместо двух иконок.** Edit + Delete заменены на компактную «⋮» с пунктами Редактировать / Дублировать / Удалить.
- **Quick-buttons частых диаметров** под слайдером пересчёта по размеру: 16/18/20/22/24/26 см. Тап — сразу выставляет диаметр (с haptic feedback).
- **Заметки на уровне секции.** В форме рецепта у каждой секции своё поле «Заметка к секции» (например, «замесить 5 минут на средней»). На экране пересчёта показывается курсивом под ингредиентами.
- **Тёмная тема** + переключатель Авто / Светлая / Тёмная (иконка в шапке списка). Предпочтение сохраняется в SharedPreferences. Базовая инфраструктура; некоторые хардкодные цвета в редких местах могут выглядеть не идеально — будут отполированы по мере находок.

## v1.10.4 — 2026-04-23

- **Hotfix:** в v1.10.3 R8 минификация ломала release-билд из-за `Missing class com.google.android.play.core.*` — Flutter ссылается на Play Core (deferred components), которого нет как зависимости. Добавлены `-dontwarn` правила в `android/app/proguard-rules.pro`. APK снова собирается.
- Содержит **все изменения v1.10.3** (тег v1.10.3 в гите есть, но release с APK не был опубликован — используйте v1.10.4).

## v1.10.3 — 2026-04-23 (release не опубликован, см. v1.10.4)

- **`versionCode` теперь монотонно растущий** (unix timestamp). Новый APK v1.10.3+ корректно ставится поверх старой версии без конфликта пакета.
- **Баннер обновления больше не показывается на iOS/desktop/web** — там APK install не работает, баннер был бесполезен.
- **`_isNewer` стал устойчив к non-strict semver** — версии вроде `1.10.2-beta` теперь сравниваются по числовой части каждого сегмента, не падают.
- **Валидация рецепта при сохранении** — пустые ингредиенты (без имени или с весом ≤ 0) и пустые секции автоматически отбрасываются. Если после очистки рецепт пустой — ошибка с понятным текстом.
- **Запятая как десятичный разделитель** — теперь можно вводить `1,5` (привычно для RU-локали), не только `1.5`. Действует на размеры формы, вес и количество ингредиентов.
- **ProGuard / R8 включены** для release-билдов (`isMinifyEnabled = true`, `isShrinkResources = true`). Должно заметно уменьшить размер APK.
- **Pre-commit hook** в `.githooks/pre-commit` — гоняет `flutter analyze` локально перед коммитом. Включается командой `git config core.hooksPath .githooks` (см. README).
- **Лицензия:** добавлен `LICENSE` (MIT).
- **+12 тестов:** 6 на `aggregateIngredients` (case-insensitivity, дедупликация, пустые имена), 6 на `StorageService` (миграция старого JSON без `notes`, backup/restore при битом main JSON, защита от полной потери данных). Всего 21 тест.

## v1.10.2 — 2026-04-23

- **Гладкая установка обновления.** `UpdateService` теперь явно запрашивает разрешение «Установка из неизвестных источников» через `permission_handler` ДО открытия APK. Раньше Android отбрасывал в Settings и не возвращал к диалогу установки. После однократного предоставления разрешения следующие апдейты проходят одним тапом без захода в настройки.
- **Осмысленные ошибки обновления.** Введён `UpdateException` с понятным текстом (например, «Нужно разрешить установку из неизвестных источников в настройках Android.»). Раньше любая ошибка показывала «Ошибка скачивания. Проверьте интернет.» — даже когда дело было не в интернете.

## v1.10.0 — 2026-04-23

- **Несколько секций одного типа** в рецепте (например, два разных крема). Раньше второй крем добавить было нельзя.
- **Заметки / шаги приготовления** — новое поле `notes` у рецепта, отображается над списком ингредиентов на экране пересчёта.
- **Поиск рецептов** по названию — появляется в шапке списка от 5 рецептов и более.
- **Список покупок** — bottom sheet с агрегированными ингредиентами (одинаковые названия суммируются по всем секциям).
- **Поделиться рецептом** — формирует текст с размерами, секциями и заметками; открывает системный share-диалог.
- **Бэкап рецептов** — `StorageService` сохраняет предыдущий валидный JSON, при битом основном — поднимает резервную копию вместо потери всех рецептов.
- **Расширены диапазоны слайдеров пересчёта**: диаметр 10–50 см (было 10–35), вес 100 г – 20 кг (было 100 г – 10 кг). Совпадает с диапазоном ручного ввода.
- **Чистка кода**: удалено мёртвое поле `_myVersion` в `main.dart` и неиспользуемый импорт.
- **README** переписан, **CHANGELOG.md** заведён.
