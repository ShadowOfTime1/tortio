# Changelog

Все значимые изменения по версиям. Формат — обратный хронологический.

## v2.6.0 — 2026-04-24

UX-оптимизация: меньше «всего сразу», проще новичку.

- **Демо-рецепт на пустом экране.** Когда рецептов нет, под «Добавьте свой первый торт!» появилась кнопка **«Создать примерный рецепт»**. По тапу — добавляется готовый шоколадный торт с 3 секциями (бисквит/крем/глазурь), заметками, тегами и оценкой. Новичок сразу видит, что и куда вписывается.
- **«Дополнительно» сворачивается** в форме рецепта. Поля **вес / заметки / оценка / теги** теперь спрятаны под `ExpansionTile` — для нового рецепта свёрнуто, при редактировании рецепта с данными — раскрыто автоматически. Новичок видит только обязательные поля: название, размер, состав.
- **«Экспорт» меню** в шапке экрана пересчёта вместо двух отдельных иконок «PDF» и «Поделиться» — одна иконка `↗` с popup'ом «Поделиться текстом / Сохранить как PDF». Меньше иконок в шапке.

## v2.5.1 — 2026-04-24

- **Тёплый градиент фона** вместо плоской заливки. Светлая тема: кремовый → нежно-розовый по диагонали. Тёмная: кофейно-чёрный → тёмно-винный. Применяется на всех экранах через тот же `DotOrnament`.
- **Орнамент крупнее** — иконки 70–110px (раньше 40–60), позиций меньше (10), спред больше. Узор «дышит» декоративно вместо мелкой ряби.
- **Карточки рецептов непрозрачные** — alpha поднят с 0.3/0.2 до 0.85/0.7 (выбранный — 1.0/0.9). Карточки теперь явно отделяются от фона, не «растворяются».

## v2.5.0 — 2026-04-24

- **Кондитерский орнамент.** Точки заменил на рассеянные иконки cake / bakery / cookie / icecream / cafe (Material Outlined) при низкой прозрачности. Тематично, не режет глаза.
- **Орнамент на ВСЕХ экранах** — главный, форма рецепта, экран пересчёта (single-tier и multi-tier). Раньше его не было на экране пересчёта.
- **«Авто» тема теперь по времени суток** (06:00–20:00 — светлая, иначе тёмная). Раньше она следовала за системой, и если телефон был в dark mode круглые сутки, приложение тоже было тёмным днём. Теперь зависит от часа, не от системы. Подпись в Settings обновлена: «Авто (день — светлая, вечер — тёмная)».
- **Авто-заполнение нового яруса.** При нажатии «Добавить ярус» диаметр предзаполняется как (предыдущий ярус − 4 см, минимум 10), высота берётся как у предыдущего. Поля редактируемые. Стандартный паттерн ярусного торта (например 30→26→22 см).

## v2.0.0 — 2026-04-24

🎂 **Multi-tier торты!** Теперь рецепт может состоять из нескольких ярусов разного размера — для свадебных тортов, ярусных, фигурных конструкций.

- **Модель:** Recipe получил поле `additionalTiers: List<TierData>`. Tier 1 живёт в существующих root-полях рецепта (`diameter`/`height`/`sections`), tier 2+ — в `additionalTiers`. Если поле пустое — рецепт одноярусный, поведение приложения идентично прошлым версиям.
- **Storage / Import-export:** дополнительные ярусы корректно сериализуются. Старые JSON без поля `additionalTiers` грузятся как одноярусные. Сохранённый JSON одноярусного рецепта НЕ содержит поле `additionalTiers` — экономим место и не пугаем старые версии клиента.
- **add_recipe форма:** под основным составом — кнопка «Добавить ещё один ярус». Каждый дополнительный ярус: метка («Низ»/«Верх»), диаметр, высота, свой состав. Можно удалить.
- **scaler_screen:** для multi-tier рецептов — отдельный экран. Каждый ярус показан как отдельная карточка со своим слайдером диаметра. Каждый ярус масштабируется **независимо** (вариант B из обсуждения). Сверху — общая инфа («N ярусов • итого ~X кг»). Список покупок и шаринг агрегируют ингредиенты со всех ярусов.
- **Дублирование рецепта** теперь корректно копирует и tier 1, и additionalTiers (ранее терялись доп.ярусы при копировании).
- **+5 unit-тестов** на multi-tier storage: round-trip, миграция старого JSON, эконом-сериализация для одноярусных.

**Известные ограничения v2.0.0:**
- В multi-tier scaler нет режима «по весу» — вес единственное число для всего торта, не масштабируется per-tier естественно. Используй размерное масштабирование.
- В additional tiers (ярус 2+) при редактировании используется упрощённый UI секций: без drag-and-drop сортировки, без autocomplete ингредиентов, без per-section заметок. Tier 1 как был — со всеми фишками. Унификация в v2.1.0.

## v1.12.0 — 2026-04-23 (release не опубликован, см. v1.12.2)

- **Теги/категории.** В рецепте — поле тегов через запятую (например, «шоколадный, без глютена»). В списке появляется лента-фильтр по тегам, всегда есть кнопка «Все» для сброса.

- **Автокомплит ингредиентов.** Поле имени ингредиента превратилось в `Autocomplete` с подсказками из истории — все когда-либо введённые названия отсортированы по частоте.
